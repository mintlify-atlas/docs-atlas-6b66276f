Loops let you repeat a block of code without writing it out multiple times. C has three loop constructs — `while`, `do-while`, and `for` — each suited to different situations. You also have `break` and `continue` to control how a loop behaves mid-execution.

## `while` loop

Runs a block of code **as long as** a condition is true. The condition is checked **before** each iteration, so if it starts out false, the body never runs.

```c
while (condition) {
    // repeated while condition is true
}
```

**Example:**

```c
int count = 0;
while (count < 5) {
    printf("count = %d\n", count);
    count++;
}
```

### Real example — the outer loop from `rpg.c`

The game spawns enemies one at a time. The outer `while` keeps the gauntlet going until either all enemies are defeated or the player runs out of HP:

```c
while (enemyCount < MAX_ENEMIES && hHp > 0) {
    eHp = 25;
    enemyCount++;

    printf("\n>>> ENEMY #%d APPEARS! <<<\n", enemyCount);

    // inner combat loop ...
}
```

Both conditions must be true at the same time (`&&`). The moment the player's HP hits zero — even mid-gauntlet — the loop ends.

## `do-while` loop

Similar to `while`, but the condition is checked **after** each iteration. This guarantees the body runs **at least once**.

```c
do {
    // runs at least once
} while (condition);
```

**Example — showing a menu that must appear at least once:**

```c
int choice;
do {
    printf("1. Attack  2. Guard  3. Quit\n");
    printf("Your choice: ");
    scanf("%d", &choice);
} while (choice < 1 || choice > 3);
```

The player always sees the menu at least once. If they enter an invalid number, it loops back and shows the menu again.

<Tip>
Use `do-while` whenever the loop body must run before the exit condition can even be evaluated — menus, input validation, and retry logic are classic use cases.
</Tip>

## `for` loop

Packs initialisation, condition check, and increment all into one line. Best when you know exactly how many times you want to iterate.

```c
for (initialisation; condition; increment) {
    // runs while condition is true
}
```

**Example:**

```c
for (int i = 0; i < 5; i++) {
    printf("Iteration %d\n", i);
}
```

You can also use multiple initialisations or increments separated by commas:

```c
for (int i = 0, j = 10; i < j; i++, j--) {
    printf("i=%d  j=%d\n", i, j);
}
```

## Comparing the three loop types

<Tabs>
  <Tab title="while">
    - Condition checked **before** each run.
    - Body may run **zero or more** times.
    - Good when you don't know upfront how many iterations you need.

    ```c
    int hp = 100;
    while (hp > 0) {
        hp -= takeDamage();
    }
    ```
  </Tab>
  <Tab title="do-while">
    - Condition checked **after** each run.
    - Body runs **at least once** — guaranteed.
    - Good for menus, input validation, retry loops.

    ```c
    int input;
    do {
        printf("Enter a positive number: ");
        scanf("%d", &input);
    } while (input <= 0);
    ```
  </Tab>
  <Tab title="for">
    - All three parts (init / condition / increment) in one line.
    - Body runs **zero or more** times.
    - Good when you know the exact number of iterations.

    ```c
    for (int i = 0; i < MAX_ENEMIES; i++) {
        spawnEnemy(i);
    }
    ```
  </Tab>
</Tabs>

## `break` and `continue`

These keywords give you fine-grained control inside any loop.

<CodeGroup>

```c break-example.c
// break exits the loop immediately
for (int i = 0; i < 10; i++) {
    if (i == 5) {
        break;  // stop at 5; never prints 5, 6, 7, 8, 9
    }
    printf("%d\n", i);
}
```

```c continue-example.c
// continue skips the rest of the current iteration
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) {
        continue;  // skip even numbers
    }
    printf("%d\n", i);  // prints 1, 3, 5, 7, 9
}
```

</CodeGroup>

### Real example — `break` in `rpg.c`

The inner combat loop in `rpg.c` uses `break` to exit as soon as the enemy is defeated or the player dies, without waiting for the `while` condition to be checked again:

```c
while (hHp > 0) {
    // ... player's turn ...

    if (eHp <= 0) {
        printf("\n[VICTORY] Enemy #%d defeated!\n", enemyCount);
        sleep(1);
        break;  // enemy dead — leave the inner loop
    }

    // ... enemy's turn ...

    if (hHp <= 0) {
        printf("\n[DEFEAT] You have fallen...\n");
        break;  // player dead — leave the inner loop
    }
}
```

Without `break`, the loop could not exit mid-iteration the moment a combatant falls.

## Nested loops

You can place one loop inside another. The inner loop runs to completion for every single iteration of the outer loop.

**Example — printing a 3×3 grid:**

```c
for (int row = 0; row < 3; row++) {
    for (int col = 0; col < 3; col++) {
        printf("[%d,%d] ", row, col);
    }
    printf("\n");
}
// Output:
// [0,0] [0,1] [0,2]
// [1,0] [1,1] [1,2]
// [2,0] [2,1] [2,2]
```

`rpg.c` uses two nested `while` loops in the same pattern: the outer loop manages the gauntlet (one iteration per enemy), and the inner loop manages a single fight (one iteration per combat turn).

## Infinite loop pattern

An intentionally infinite loop runs until something inside it calls `break` or `return`. This is common in game loops, server event loops, and menu-driven programs.

```c
while (1) {
    int choice = getPlayerInput();
    if (choice == QUIT) {
        break;
    }
    processChoice(choice);
}
```

`while (1)` is the conventional way to write it in C. Some developers also write `for (;;)` for the same effect.

## Common mistakes

<Warning>
**Off-by-one errors** are the most frequent loop bug. They happen when a loop runs one too many or one too few times.

```c
// Intends to print indices 0 through 4 (5 items)
// BUG: uses <= instead of <, so it also runs for i == 5
for (int i = 0; i <= 5; i++) {
    printf("%d\n", i);  // prints 0, 1, 2, 3, 4, 5 — one extra!
}

// Correct:
for (int i = 0; i < 5; i++) {
    printf("%d\n", i);  // prints 0, 1, 2, 3, 4
}
```
</Warning>

<Warning>
**Forgetting to increment** turns a `while` loop into an infinite loop that never makes progress.

```c
// BUG: count is never incremented
int count = 0;
while (count < 5) {
    printf("%d\n", count);
    // forgot count++; — loops forever!
}

// Correct:
int count = 0;
while (count < 5) {
    printf("%d\n", count);
    count++;  // advances the loop
}
```
</Warning>
